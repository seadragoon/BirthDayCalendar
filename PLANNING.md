# 目的
Flutterを用いて、Yahoo!カレンダー（Y!カレンダー）のUI/UXをベースとした、シンプルかつ拡張性の高いカレンダーアプリを作成してください。

# 技術スタック
- **Framework:** Flutter
- **Language:** Dart (Null Safety)
- **State Management:** Riverpod (flutter_riverpod)
- **Calendar Library:** calendar_view (数日に跨ぐバー表示対応のため)
- **Database:** sqflite, path (ローカル保存用)
- **Others:** intl (日付・多言語処理)

# 環境構築の指示
1. プロジェクト名 `birthday_calendar` としてFlutterプロジェクトを初期化してください。
2. `pubspec.yaml` に `calendar_view`, `flutter_riverpod`, `intl`, `sqflite`, `path` を追加し、`flutter pub get` を実行してください。
3. Androidの `minSdkVersion` を 21 以上に設定してください。
4. アセットディレクトリ `assets/images/themes/` を作成し、背景画像読み込みのパスを通してください。

# アーキテクチャと設計指針
### 1. ディレクトリ構成 (Feature-based)
機能ごとにディレクトリを分け、UI、状態管理、データモデルを別ファイルで作成してください。
- `lib/features/calendar/`: スケジュール関連
- `lib/features/birthday/`: 誕生日関連
- `lib/features/settings/`: 設定・きせかえ関連
- `lib/shared/`: 共通部品、DBヘルパー、テーマ

各機能内は以下の層に分割してください：
- `models/`: データ型定義
- `repositories/`: データ操作（DataRepositoryインターフェースの実装）
- `providers/`: Notifier/AsyncNotifierによる状態管理
- `views/`: ConsumerWidget/ConsumerStatefulWidgetによるUI

### 2. 設計方針
- **疎結合:** UIは直接DBに触れず、Providerを介してRepositoryを操作する構成にすること。
- **メンテナンス性:** ロジックをWidget内に書かず、Notifierに集約すること。

# 機能概要
- Yahooのカレンダーアプリ「Y!カレンダー」をベースに下記機能を備えるカレンダーアプリ。
- 月間カレンダーの表示機能。
- 日付をタップした際の選択状態の可視化。
- 選択された日付に紐づくイベント（予定）のリスト表示。
- イベントの表示・追加・編集機能。
- 誕生日の表示・追加・編集機能。（誕生日もイベントとして扱う）

# 実装してほしい動作
### 1. データ設計
- **EventModel:** title, startDate, endDate, isAllDay, colorIndex, recurrence, notification, comment, isBirthday(bool)
- **BirthdayModel:** name, date, isYearUnknown, tags(List), notification
- **保存形式:** `DataRepository`インターフェースを定義し、初期実装として`sqflite`でCRUDを作成。

### 2. UIの構成と実装（デザイン参考：sample-ui/yahoo.png）
- **Header:** タイトル、メニュー、検索、今日ボタンを配置。表示中のViewに応じてタイトルとボタンを切り替え。
- **Main View:** - **Schedule:** `MonthView`を使用。5週/6週の高さ調整、日本語(ja_JP)対応、左右スライドで月切り替え、日付跨ぎのバー表示、下部にイベントリスト。
    - **Birthday:** タグ（家族/友達/カスタム）によるフィルタリング、満年齢の表示。
- **Footer:** 「スケジュール」「誕生日」の切り替えボタン。背景画像の適用に対応。
- **FAB:** 右下に配置。現在のViewに合わせて「予定追加」か「誕生日追加」のモーダルを開く。
- **Full Screen Modal:** 共通のBaseModalヘッダー（閉じる、決定、削除、編集ボタンの切り替え）を使用。
    - **検索:** 文字入力中のリアルタイム検索、予定と誕生日の横断表示。
    - **イベント追加/編集:** 開始・終了日のバリデーション、12色選択、通知設定（システム設定状況の表示）。
    - **誕生日追加/編集:** 生まれ年不明チェック、複数タグ選択、カスタムタグ追加機能。

### 3. UIの構成と実装の詳細

#### 全体方針
- **基本レイアウト:** スマートフォン向けUI（横幅いっぱいを使用）。
- **画面方向:** 縦向き固定（横向き表示には非対応）。
- **デザイン参考:** `sample-ui/yahoo.png` のデザインをベースとする。
- **きせかえ機能:** - Header/Footerに画像を適用可能にする。
  - Main Viewには、背景画像をホワイトアウトさせた状態で設定可能にする。
  - 画像は固定パス（`assets/images/themes/`等）以下に配置した3種類の画像から読み込む設計とする。

#### Header View
- **配置:** 画面最上部に固定。
- **構成:** - 左端：メニューボタン（ドロワー表示）。
  - 中央：タイトル文字列。
  - 右端：検索ボタン。
  - 検索ボタンの左隣：「今日」ボタン。
- **動的表示:**
  - Schedule View時：タイトルに「2026年4月」等の年月を表示。
  - Birthday View時：タイトルに「誕生日」と表示。
  - 表示タイプ毎にタイトルやボタン構成を外部から注入・切り替え可能な汎用的な設計にすること。
- **メニュー内容:** - 表示切り替え（チェックボックス形式。誕生日のON/OFF等）。
  - 最下部に設定アイコンを配置。

#### Main View
表示状態（Schedule / Birthday）を切り替えて表示。

##### A. Schedule View
- **構成:** Calendar View と Event List を縦方向に連結。
- **Calendar View:**
  - `calendar_view`を使用し、月間表示をデフォルトとする。
  - 日本語対応（locale: 'ja_JP'）。上部に「日～土」の曜日を表示。
  - 左右スワイプで月移動が可能。
  - 5週表示と6週表示で1週間の高さが同一になるよう調整すること。
  - 前月・来月の日付も薄く表示。
  - **イベント表示:** 各日の予定は表示可能な限り「帯（バー）」で表示。日付を跨ぐ予定は連続した帯として表示すること。
  - タップで「選択中の日付」を切り替え。
- **Today Bar:**
  - 選択中の日付（例：「4月6日(月)」）を1列分表示。
- **Event List:**
  - 選択中の日付に紐づくイベントを `ListView.builder` で表示。
  - タップで「イベント表示モーダル」へ遷移。
  - 予定がない場合は「予定はありません」のプレースホルダーを表示。

##### B. Birthday View
- **Tag Filter:**
  - 上部に「すべて」「家族」「友達」「カスタムタグ...」「未設定」を横並びで表示。
  - タップでチェック状態を切り替え、該当するデータのみをリスト表示。
- **List View:**
  - 名前と誕生日を表示。
  - 誕生年が判明している場合は、右側に「満xx歳」を表示。
  - タップで「誕生日追加編集モーダル」へ遷移。

#### Footer
- **構成:** 画面最下部にアイコンとラベルのセットを配置。
- **ボタン:** 「スケジュール」「誕生日」を左側に配置。右側は将来の拡張用に空けておく。

#### Floating Action Button (FAB)
- **配置:** 画面右下に独立して配置。
- **動作:** - Schedule View時は「イベント追加モーダル」を表示。
  - Birthday View時は「誕生日追加編集モーダル」を表示。

#### Full Screen Modal (共通基盤)
画面全体を覆う共通デザインのモーダル。
- **共通Header:** - 左端：閉じる（×）ボタン。
  - 中央：メニュー名（検索時は検索バー）。
  - 右端：決定 または 削除（ゴミ箱） または 編集（鉛筆）ボタン。

##### 各モーダルの詳細
1. **検索モーダル:**
   - 全予定/誕生日を対象に検索。リアルタイム検索に対応。
   - フィルタ（予定のみ/誕生日のみ）をチェックボックスで切り替え可能。
2. **設定モーダル:**
   - 「アプリ設定」「きせかえテーマ」「ライセンス」「バージョン」をリスト表示。
3. **イベント表示モーダル:**
   - 内容：タイトル、開始/終了日付、繰り返し、色、通知、コメント。
   - 右上の削除ボタン押下で確認ダイアログ（Yes/No）を表示。
4. **イベント追加/編集モーダル:**
   - 項目：タイトル（必須）、開始/終了日付（Picker使用）、終日トグル、繰り返し、色（12色Enum）、通知、コメント。
   - バリデーション：開始 > 終了にならないよう自動調整。
5. **誕生日追加/編集モーダル:**
   - 項目：名前（必須）、日付（必須）、生まれ年不明チェック、通知、タグ。
   - 生まれ年不明時は年の入力・表示を不要とする。
   - タグは複数選択可能。新規タグ追加用ダイアログへの導線を用意。
